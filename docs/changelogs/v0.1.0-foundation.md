# Prompts Foundation v0.1.0

Initial foundation implementation for SDKWork forum module.

## Date

2026-06-13

## Summary

Complete foundation skeleton with database contracts, API contracts, SDK family metadata, route manifests, Rust module boundaries, TypeScript SDK facades, tests, deployment configs, and operational documentation.

## Database Schema

- 45 table contracts in `specs/forum-database.schema.yaml`
- 8 table groups: taxonomy, discussion, qa_poll, engagement, member, moderation, projection, integration
- Standard field sets: tenant_entity, integration_log
- Full constraint and index definitions

## API Contracts

- **App API** (`/app/v3/api/forum`): 22 routes for user-facing forum operations
- **Backend API** (`/backend/v3/api/forum`): 36 routes for admin/moderation operations
- **Open API** (`/prompts/v3/api`): 8 routes for anonymous public reads

## Migration Notes

No migrations generated yet. Database schema is defined in YAML contract. SQL migrations will be generated when the schema generator is connected.

## Known Limitations

- Repository layer returns `not_implemented` errors (requires SQLx pool injection)
- Integration ports use Noop implementations (requires external SDK dependencies)
- Worker jobs return `Ok(())` without side effects (requires SQLx pool connection)
- No OpenAPI YAML files yet (requires route manifest materialization)
